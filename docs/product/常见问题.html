常见问题

什么是企业应用连接器？#

企业应用连接器 = 千帆 IDaaS（账号组织连接）+千帆 iPaaS（接口数据连接）+千帆 aPaaS（应用及业务连接）。

  • 账号组织连接-千帆 IDaaS：面向 SaaS 伙伴的身份&账号连接器。
  • 接口数据连接-千帆 iPaaS：一站式企业应用数据连接器。
  • 应用及业务连接-千帆 aPaaS：像做幻灯片一样构建企业应用。

什么是 aPaaS？#

aPaaS 的全称是 application Platform as a Service，即应用程序平台即服务。Gartner 对其所下的定义是：“这是基于 PaaS（平台即服务）的一种解决方案，支持应用程序在云端的开发、部署和运行，提供软件开发中的基础工具给用户，包括数据对象、权限管理、用户界面等。”

aPaaS 和 PaaS 的区别是什么？#

aPaaS 和 PaaS 都可以完成软件的开发和部署，都支持云端访问。而两者的差异主要体现在用户人群和使用环境不一样：

  • PaaS 包含所有平台级别的服务，需要技术人员在本地完成应用程序的开发和数据提供，然后部署到 PaaS 平台上，再分发给用户使用。
  • aPaaS 是 PaaS 的一种子形式，在 aPaaS 模式下，非技术人员可以直接在云端完成应用程序的搭建、部署、使用、更新和管理。

PaaS 比较灵活，可以完全按需开发，但是人力成本较高，需要招募程序员去完成开发过程；另外，如果涉及到需求的调整，还会涉及到二次开发。不管是人力成本和时间都不是一般企业能够承受的。aPaaS 的优点是快、成本低，非技术人员即可开发一套商业软件，在开发过程中可见即可得，方便随时调整。

什么是千帆 aPaaS？#

千帆 aPaaS 是一款专注于企业级应用可持续开发管理的模型驱动、可视化、集成开放的高效能云上开发平台，解决传统代码开发模式成本高昂，开发的速度赶不上用户需求速度，且灵活性、敏捷性差等问题。企业可以通过千帆 aPaaS 在可视化的配置界面进行简单的拖拽、组合标准插件、高效搭建应用系统，有效降低技术门槛，大大提高产品交付效率，帮助企业节省大量人力及时间成本。

具备如下特征：

  • 可视化应用设计：可视化数据建模、页面设计、流程编辑，所见即所得（WYSIWYG）的设计效果。
  • 低代码扩展：支持少量代码、脚本灵活自定义扩展，如流程处理逻辑、UI 样式、页面动作等，以及与外部系统集成连通。
  • 全流程管理：一站式应用开发、管理平台，包涵设计、测试、发布、运维监控等。
  • 多环境支持：支持生产、沙盒多种环境的部署，满足企业复杂应用多环境测试的诉求。
  • 灵活人员权限管理：灵活控制不同身份的访问不同的功能和数据，做到信息高速传递的同时，保护信息安全。
  • 开放集成能力：开放 OpenAPI、SDK、集成 iPaaS 连通企业应用生态。
  • 数据分析能力：支持 BI 数据分析，自定义折线图、柱状图、饼图等数据分析看板。

千帆 aPaaS 底层是什么技术栈？#

前端：react；后端：java，部分 nodejs。

应用是否部署在云端的？#

目前是部署在云端。

用户量大时千帆 aPaaS 是否会有性能瓶颈？#

性能有保障，底层基于腾讯云平台，有对应技术支持。

使用千帆 aPaaS 交付能提升多少效率？#

根据项目的复杂情况会不同，总统上会大幅提升交付效率。

千帆 aPaaS 是否支持自定义代码开发？#

支持，例如：自定义组件、自定义页面动作、UI 样式、流程节点自定义等。通过 JS 代码、自定义表达式以及云函数的形式实现个性化开发。

千帆 aPaaS 支持哪些应用搭建的场景？#

可从如下角度理解：

  • 从应用的功能边界区分为全新应用和增量模块
  • 全新应用：从0～1搭建全新应用，该应用可与外部数据或系统数据集成交互。
  • 增量模块：已有的 SaaS 应用如需新增模块，可直接在前方 aPaaS 平台快速搭建，并支持与已有功能做无缝数据集成和页面集成，让终端用户无感知。
  • 从应用的访问端形态分为 Web 端应用和小程序应用。
  • Web 端应用：支持一次设计多端适配，即同一个应用在 PC 和移动均可访问，同时也支持企业微信工作台应用。
  • 小程序应用：支持创建小程序，并一键发布（建设中）。

为什么申请千帆 aPaaS 内测被驳回？#

千帆 aPaaS 目前正在内测中，会定向邀请与当前平台定位和服务能力吻合度高的客户试用，待公测后，会邀请您再试用。

千帆 aPaaS 平台价值有哪些？#

  • 成本低：应用开发成本降低至少70%，较传统开发相比，通过拖拉拽即可完成应用的基本开发，无需专业的前后端开发技术。
  • 交付快：会画图就会设计应用，缩短企业应用软件定制需求的交付周期，提高开发人员的开发速度，业务人员也可以直接学习并参与构建应用，人人都可以是开发者。
  • 更强大：数据模型驱动应用，分离数据设计、流程设计、页面设计，支持高度复杂的企业内部以及 C2B、C2G 相关应用。
  • 可扩展：企业可以随时新建应用，调整数据存储结构、流程、页面，客户业务变化，应用快速跟着变化，应用版本升级以及二次开发灵活扩展。
  • 可复制：支持将实施项目升级为模板应用，复制率可达到80%，避免每个项目都全部重写代码，可快速在多个项目间复用已经定义好的数据模型、流程和页面，轻松交付项目。
  • C2B 连接：帮助客户和合作伙伴，更好地服务 C 端用户。全面融合微信、QQ、企业微信的连接能力，让社会和企业服务更高效地触达用户，通过小程序、公众号、微信支付等连接工具，保证服务能力和用户体验。
